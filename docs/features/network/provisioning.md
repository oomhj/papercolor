# 配网流程设计

PaperColor 的 WiFi 配置方案 —— 不需要 App，不依赖屏幕交互。

---

## 一、核心思路

设备没有触屏，纯靠 3 个按键无法完成 WiFi 输入。因此配网通过**外部设备**（手机/电脑）完成：

```
用户操作                        PaperColor
─────────                      ──────────
                          开机 → NVS 无 WiFi 配置？
                                        │
                           ┌── YES ─────┼──── NO ──┐
                           │            │           │
                        连接 STA   启动 AP 热点     │
                           │         + DNS 劫持     │
                           │            │           │
                       ┌─ 成功 ─┐       │           │
                       │      失败      │           │
                  进入应用   启动 AP     │           │
                                   │              │
                    ─── 手机连接 PaperColor 热点 ───┘
                    │
             打开浏览器 → 自动弹出配网页
                    │
              输入 WiFi 账号密码
                    │
              提交 → 保存到 NVS
                    │
              设备重启 → 自动连接 STA
```

---

## 二、AP 热点配网流程（主方案）

### 2.1 启动 AP

设备在没有 WiFi 配置或 STA 连接失败时，自动启动 AP 热点：

| 参数 | 值 |
|------|-----|
| SSID | `PaperColor-XXXX`（XXXX = MAC 后 4 位） |
| 密码 | 无（开放网络） |
| IP | `192.168.4.1` |
| 信道 | 6 |

### 2.2 DNS 劫持

启动 DNS 服务器，将所有域名请求都解析到 `192.168.4.1`：

```
用户访问 http://example.com
    → DNS 返回 192.168.4.1
    → 浏览器显示设备内置的配网页
```

这样用户连接热点后，随便打开一个网址都会弹出配网页（Captive Portal）。

### 2.3 配网页

单页 HTML，内嵌在固件中：

```
┌────────────────────────────────────┐
│  📶 PaperColor WiFi Setup         │
│                                    │
│  SSID:  [________________]   🔍   │  ← 扫描按钮
│  Password: [________________]      │
│                                    │
│  [        Connect        ]         │
│                                    │
│  Status: Waiting...                │
│  ─────────────────────────         │
│  Hotspot: PaperColor-A84A          │
│                                    │
│  Tips: 仅支持 2.4GHz WiFi          │
└────────────────────────────────────┘
```

### 2.4 配网 API

| 端点 | 方法 | 说明 |
|------|------|------|
| `/` | GET | 配网页 HTML |
| `/api/scan` | GET | 扫描附近 WiFi（返回 JSON 列表） |
| `/api/config` | POST | 提交 WiFi 配置 `{"ssid":"...", "pass":"..."}` |
| `/api/status` | GET | 查询连接状态 |

**`POST /api/config` 响应：**
```json
{"status": "ok", "message": "Connecting..."}
```

**`GET /api/status` 响应：**
```json
{"connected": true, "ip": "192.168.1.100"}
```

### 2.5 AP 自动关闭

| 条件 | 行为 |
|------|------|
| STA 连接成功 | AP 保持 3 分钟，无客户端则关闭 |
| 有 AP 客户端连接 | 延迟关闭，持续提供配网 |
| 配网成功 | 重启设备，AP 消失 |
| 超时 10 分钟无操作 | 关闭 AP，进入低功耗待机 |

## 三、物理按键触发配网

任何时候，用户可以通过按键重新进入配网模式：

```
当前状态                    操作
─────────                  ──
正常运行 (STA 已连接)      长按 BTN-A 5 秒
                            → LED 变绿
                            → 启动 AP 热点
                            → 恢复配网页
                            → 用户可重新配置 WiFi
```

这样换网络时不需要重新编译烧录。

---

## 四、多设备漫游

NVS 中可保存**多组** WiFi 配置（最多 3 组）：

```
wifi_ssid_0, wifi_pass_0    ← 首选
wifi_ssid_1, wifi_pass_1    ← 备选 1
wifi_ssid_2, wifi_pass_2    ← 备选 2
```

连接时按优先级尝试，全部失败则进入 AP 配网模式。

## 五、状态机

```
          ┌─────────────────────────────────────┐
          │                                     │
          ▼                                     │
    ┌──────────┐    AP 超时     ┌──────────┐    │
    │ AP_IDLE  │ ────────────► │ SLEEP    │    │
    └────┬─────┘               └──────────┘    │
         │ 手机连接热点                          │
         ▼                                     │
    ┌──────────┐    用户提交     ┌──────────┐   │
    │ AP_CFG   │ ────────────► │ STA_CN   │   │
    └──────────┘               └────┬─────┘   │
                                    │         │
                              ┌─── 成功 ─┐    │
                              │        失败   │
                              ▼         │    │
                         ┌────────┐     │    │
                         │ ONLINE │     └────┘
                         └────────┘
                              │
                         断线/按键
                              │
                              ▼
                         ┌────────┐
                         │ RECON  │── 重试 3 次
                         └───┬────┘
                             │ 全失败
                             ▼
                         回到 AP_IDLE
```

## 七、LED 状态指示

| 状态 | LED 效果 | 说明 |
|------|----------|------|
| 启动中 | 熄灭 | 硬件初始化 |
| STA 连接中 | 蓝灯慢闪 (1s 周期) | 正在连接路由器 |
| STA 已连接 | 绿灯亮 0.5s 后熄灭 | 连接成功 |
| STA 失败 | 红灯亮 2s 后熄灭 | 重试或进入 AP 模式 |
| AP 待机 | 蓝灯快闪 (200ms) | 热点已开，等待客户端 |
| AP 配网中 | 蓝灯常亮 | 有人在配网页操作 |
| 配网成功 | 绿灯闪烁 3 次 | 配置已保存，即将重启 |
| 休眠 | 熄灭 | 深度睡眠 |
