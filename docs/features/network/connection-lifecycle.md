# 连接生命周期管理

WiFi 连接的自动重试、失败处理、手动触发配网 — 当前实现说明。

---

## 一、连接状态机

```
         ┌──────────┐
         │   OFF    │  WiFi 未初始化
         └────┬─────┘
              │ wifi_mgr_init()
         ┌────▼─────┐
         │  INIT    │  ← 系统启动，注册事件处理器
         └────┬─────┘
              │ wifi_mgr_start_retry_loop()
         ┌────▼─────┐
         │ STA_CN   │  STA 连接中（按优先级尝试保存的网络）
         └────┬─────┘
              │
        ┌─────┴────────┐
        │              │
    ┌───▼───┐    ┌────▼────┐
    │STA_OK │    │STA_FAIL │  ← 全部尝试失败
    └───┬───┘    └────┬────┘
        │             │
    ┌───▼───┐    ┌────▼────┐
    │STA_LOST│    │AP_IDLE  │  ← AP 热点已开启
    └───┬───┘    └────┬────┘
        │             │ 手机连接热点
        │         ┌────▼────┐
        │         │ AP_CFG  │  ← 用户在配网页操作
        │         └────┬────┘
        │              │ 提交 WiFi 配置
        │         ┌────▼────┐
        └── 重试 ─┤STA_CN   │  ← 连接新配置
                 └─────────┘
```

## 二、连接策略

### 2.1 重试参数

```
尝试第 1 次：超时 10s，等待 0s（立即）
  ├─ 各 Slot 逐一尝试，每 slot 等待 10s
  ├─ 成功 → STA_OK（同时启动 AP 并保持 3 分钟供用户配置）
  └─ 失败 → 等待 5s

尝试第 2 次：超时 10s，等待 5s
  ├─ 各 Slot 逐一尝试，每 slot 等待 10s
  └─ 失败 → 等待 15s

尝试第 3 次：超时 15s，等待 15s
  ├─ 各 Slot 逐一尝试，每 slot 等待 15s
  └─ 全部失败 → 启动 AP 配网模式
```

### 2.2 运行时断线重连

WiFi 断开事件（`WIFI_EVENT_STA_DISCONNECTED`）触发状态从 `STA_OK` 变为 `STA_LOST`。`wifi_mgr_start_retry_loop(true)` 使用相同退避策略：

| 尝试次数 | 等待时间 | 超时 | 呼吸灯亮度范围 |
|----------|----------|------|---------------|
| 1 | 0s（立即） | 10s | 10–120 |
| 2 | 5s | 10s | 5–80 |
| 3 | 15s | 15s | 3–50 |
| 之后 | 进入 AP 配网 | — | — |

### 2.3 呼吸灯效果

连接过程中 LED 使用呼吸灯效果：

- **连接中（慢速呼吸）**：2s 周期蓝色呼吸灯
  - 第 1 次重试：较亮（10–120），表示稍有信心
  - 第 2 次重试：中等（5–80），信心降低
  - 第 3 次重试：很暗（3–50），即将放弃
- **连接成功（绿灯 3 秒）**：绿灯常亮 3 秒后熄灭，然后进入 AP 配网模式
- **配网模式（快速呼吸）**：500ms 周期蓝色快闪呼吸灯，直观提示设备处于配网等待状态

## 三、手动触发配网

用户在运行中可通过按键主动进入配网模式：

### 操作方式

| 操作 | 当前状态 | 结果 |
|------|----------|------|
| **长按 BTN-B (G9) 3 秒** | 任何状态 | 断开 STA → 启动 AP + 配网页 |
| **长按 BTN-C (G1) 5 秒** | 任何状态 | 深层休眠 |

### 按键定义

| 按键 | GPIO | M5Unified 对象 | 功能约定 |
|------|------|----------------|----------|
| BTN-TOP (C) | G1 | `M5.BtnC` | 顶部按键，长按 5s 休眠 |
| BTN-UP (B) | G9 | `M5.BtnB` | 中间按键，长按 3s 进入配网 |
| BTN-DOWN (A) | G10 | `M5.BtnA` | 底部按键，各应用自定功能 |

### LED 反馈

| 操作 | LED 效果 |
|------|----------|
| 按下 BTN-B | 蓝灯亮 60 亮度 |
| 3 秒到达 | 绿灯亮 200ms，AP 启动 |
| AP + 配网已启动 | 蓝灯亮 （`WIFI_STATE_AP_IDLE`） |
| 客户端连接 AP | 青灯亮（`WIFI_STATE_AP_CFG`） |

## 四、配网完成后的行为

用户通过配网页提交 WiFi 配置后：

```
1. 保存到 NVS slot 0
2. 返回 HTTP 200 {"status":"ok"}
3. 延时 500ms（让浏览器收到响应）
4. 重启设备（esp_restart()）

重启后：
  ├─ wifi_mgr_start_retry_loop(false)
  ├─ 尝试保存的配置 → 成功 → STA_OK
  └─ 尝试失败 → 启动 AP 配网
```

注意：连接成功后 AP 和配网服务器**不会立即关闭**，而是保持 **3 分钟**，让用户仍然可以通过浏览器配置网络中已保存的其他网络。

## 五、API 参考

```c
/** @brief 开始连接重试循环（含退避，最多 3 次）*/
void wifi_mgr_start_retry_loop(bool is_reconnect);

/** @brief 停止重试 */
void wifi_mgr_stop_retry(void);

/** @brief 手动触发配网模式（由按键回调调用） */
void wifi_mgr_trigger_provisioning(void);

/** @brief 重启设备 */
void wifi_mgr_restart(void) __attribute__((noreturn));

/** @brief 在主循环中调用，检测按键触发配网/休眠 */
bool wifi_mgr_handle_buttons(void);

/** @brief 根据当前状态设置 LED 颜色 */
void wifi_mgr_update_led(void);
```

## 六、LED 状态总表

| 状态 | LED | 触发 |
|------|-----|------|
| OFF / INIT | 熄灭 | 系统启动中 |
| STA 连接中 (STA_CN) | 蓝灯慢速呼吸（2s 周期，亮度递减） | 等待连接 |
| STA 已连接 (STA_OK) | 绿灯常亮 3 秒后熄灭 | 连接成功 |
| STA 断开 (STA_LOST) | 橙灯 | 运行时断线 |
| STA 失败 (STA_FAIL) | 红灯 | 全部重试失败 |
| AP 待机 (AP_IDLE) | 蓝灯快速呼吸（500ms 周期快闪） | AP 开启，等待客户端配置 |
| AP 配置中 (AP_CFG) | 青灯常亮 | 客户端正在操作配网页 |
| 休眠 | 熄灭 | 深层睡眠 |
