# 网络相册 — 设计方案

在 PaperColor 墨水屏上滚动显示网络图片。

---

## 一、数据源方案

### 方案 A：固定 URL 列表（P0）
```c
// 硬编码一组图片 URL
"https://picsum.photos/600/400"  // 每次返回随机图片
"https://bing.com/..."           // Bing 每日一图
```
- 简单直接，无需服务器
- 每张图需要单独 HTTP 请求

### 方案 B：JSON 清单（P1）
设备从固定 URL 拉取 JSON 清单，清单里列出所有图片 URL：
```json
[
  {"url": "https://.../photo1.jpg", "title": "标题"},
  {"url": "https://.../photo2.jpg", "title": "标题"}
]
```
- 灵活，可远程更新
- 需要维护一个 JSON 文件（GitHub/Gist 即可）

### 方案 C：自建 Web 服务推送
设备运行 HTTP server（已有 `app_server` 基础），手机/电脑上传图片。
- 最灵活
- 实现最复杂

**推荐：P0 用方案 A，P1 升级到方案 B**

## 二、EPD 显示布局

横屏 600×400：

```
┌──────────────────────────────────────────────────────────────────┐
│  ◀  Album                           3/12     2026-07-03   ██   │  ← 顶栏 h=32
├──────────────────────────────────────────────────────────────────┤
│                                                                  │
│                          ┌──────────┐                            │
│                          │          │                            │
│                          │  图片     │                            │
│                          │  自适应    │                            │  ← 图片区域
│                          │  缩放居中  │                            │    填充可用空间
│                          │          │                            │
│                          └──────────┘                            │
│                                                                  │
├──────────────────────────────────────────────────────────────────┤
│  [A] ◄ Prev     [B] Refresh     [C] Next ►                      │  ← 底栏 h=30
└──────────────────────────────────────────────────────────────────┘
```

### 图片处理
- 源图缩放到 EPD 尺寸时保持宽高比
- 空白区域用纯色填充
- 全彩模式（8-bit）

## 三、滑动动画

EPD 不适合帧率动画，但可以做**步进滑动**：

```
┌──────────────────────┐    ┌──────────────────────┐
│ 旧图片   │  空白      │ →  │  空白   │  新图片     │
│  (左半)  │  (右半)    │    │ (左半)  │  (右半)     │
└──────────────────────┘    └──────────────────────┘
```

实现方式：
1. 使用 `epd_fastest` 模式（低质量但刷新快）
2. 每次步进 30px，分 ~10 步完成
3. 每步调用 `display()` 局部刷新

但注意：频繁刷新 EPD 会降低寿命，可增加一个配置开关 `animation: on/off`。

## 四、技术关键点

| 问题 | 方案 |
|------|------|
| **图片解码** | M5GFX 内置 `drawJpg()` / `drawPng()`，直接从内存渲染 |
| **大图内存** | JPEG 约 50-200KB，8MB PSRAM 足够缓冲多张 |
| **下载超时** | 30s 超时，失败显示占位图 |
| **WiFi 重连** | 每次翻页/刷新时连接，完事后可断开 |
| **HTTPS 证书** | 复用 news_fetcher 的 `crt_bundle_attach` |
| **EPD 刷新限制** | 翻页间隔 ≥ 5s，避免频繁刷写 |

## 五、用户体验流程

```
开机 → 白屏 → 连接 WiFi(蓝灯) → 下载图片(蓝灯) → 显示图片(绿灯 0.5s)
                                                      ↓
                                              [A] Prev  ←  →  [C] Next
                                                      ↓
                                              下载下一张 → 滑动动画 → 显示
```

## 六、文件结构

```
main/apps/album/
├── album_app.h         # 应用生命周期
```

- `apps/news/news_fetcher`（复用 HTTP 下载模块）

## 七、实施路线
