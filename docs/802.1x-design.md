# 802.1x (PEAP-MSCHAPv2) 支持设计

## 概述

为 PaperColor 添加 WPA2-Enterprise 网络支持，使用 PEAP + MSCHAPv2 认证方式。
跳过服务器证书验证，不涉及 CA / 客户端证书，核心只需要 identity + username + password 三个新字段。

## 关键发现：TLS 1.0 兼容性

某些企业 AP（如华为）的 PEAP 隧道只支持 TLS 1.0。

```
Mac Client Hello:  Version TLS 1.2 (0x0303)
Server Hello:      Version TLS 1.0 (0x0301), cipher TLS_RSA_WITH_AES_128_CBC_SHA
Mac 接受降级 → 握手成功
```

ESP-IDF v5.5 使用 **mbedTLS 3.x**，该版本已完全移除 TLS 1.0/1.1 代码。
使用 mbedTLS 时，ESP32 会直接回复 `TLS Fatal Alert: Protocol Version`，不会尝试 Client Hello。

### 解决方案：wpa_supplicant 内部 TLS

ESP-IDF 的 wpa_supplicant 组件内置了独立的 TLS 实现（`tlsv1_client.c`），
支持 TLS 1.0/1.1/1.2。通过关闭 `ESP_WIFI_MBEDTLS_TLS_CLIENT` 选项启用。

同时需关闭 `ESP_WIFI_MBEDTLS_CRYPTO`（使用 wpa_supplicant 内部加密），
并开启 `ESP_WIFI_ENTERPRISE_SUPPORT`。

## 构建配置

### Kconfig 手动修改（必须）

每次重新创建编译容器后，需修改 ESP-IDF 的 Kconfig：

```bash
# 1. ESP_WIFI_ENABLED — 默认开启 WiFi（ESP32-S3 有 WiFi 硬件）
sed -i 's/default n if SOC_WIFI_SUPPORTED$/default y if SOC_WIFI_SUPPORTED/' \
  /opt/esp/idf/components/esp_wifi/Kconfig

# 2. ESP_WIFI_MBEDTLS_CRYPTO — 关闭 mbedTLS 加密，使用内部加密
sed -i '/config ESP_WIFI_MBEDTLS_CRYPTO/{n;s/default y/default n/}' \
  /opt/esp/idf/components/esp_wifi/Kconfig

# 3. ESP_WIFI_MBEDTLS_TLS_CLIENT — 关闭 mbedTLS TLS，使用内部 TLS
sed -i '/config ESP_WIFI_MBEDTLS_TLS_CLIENT/{n;n;s/default y/default n/}' \
  /opt/esp/idf/components/esp_wifi/Kconfig
```

原因：这三个选项的 Kconfig `default y` 无法通过 `sdkconfig.defaults` 覆盖。

### sdkconfig 附加选项（在 sdkconfig 或 defaults 中设置）

```
CONFIG_ESP_WIFI_ENTERPRISE_SUPPORT=y
```

### 组件依赖（CMakeLists.txt）


## Enterprise 连接逻辑

### 连接步骤（精确顺序）

参考 ESP-IDF 官方例程 `wifi_enterprise_main.c`，特殊处理 Enterprise 路径：

```c
// PSK 路径（保持原有行为）
esp_wifi_set_mode(WIFI_MODE_STA);
esp_wifi_set_config(WIFI_IF_STA, &wc);   // 含 ssid + password
esp_wifi_start();                         // STA_START → connect

// Enterprise 路径
esp_wifi_set_mode(WIFI_MODE_STA);
esp_wifi_set_config(WIFI_IF_STA, &wc);   // 仅 ssid，不设 password/authmode

// EAP 参数必须在 enterprise_enable + start 之前设置
esp_eap_client_set_identity(...);
esp_eap_client_set_username(...);
esp_eap_client_set_password(...);
esp_eap_client_set_eap_methods(ESP_EAP_TYPE_PEAP);
esp_eap_client_set_disable_time_check(true);

esp_wifi_sta_enterprise_enable();
esp_wifi_start();                         // STA_START → connect（EAP 已就绪）
```

注意：
- 不设置 `wifi_config_t.sta.password`（EAP 客户端独立处理密码）
- 不设置 `wifi_config_t.sta.threshold.authmode`（wpa_supplicant 自动检测）
- `esp_wifi_sta_enterprise_enable()` 必须在 `esp_wifi_start()` 之前

### 断开处理

```c
void wifi_mgr_disconnect_sta(void)
{
    esp_wifi_sta_enterprise_disable();  // 清除 EAP 状态
    esp_wifi_disconnect();
    esp_wifi_set_mode(WIFI_MODE_NULL);
}
```

### LED 自动更新

每次 WiFi 状态变更时自动调用 `wifi_mgr_update_led()`（在 `set_state()` 中）：

| 状态 | LED | 含义 |
|------|-----|------|
| STA_CN | 蓝呼吸 | 连接中 |
| STA_OK | 绿灯 | 已连接 |
| STA_FAIL | 红闪 3× | 连接失败 |
| STA_LOST | 橙闪 10× | 运行中断开 |
| AP_IDLE | 黄闪 | 配网等待客户端 |
| AP_CFG | 青灯 | 客户端正在配置 |

## 配网流程

详细的完整流程：

```
用户 UP+DOWN
  ↓
读取 SD config.txt → 写入 NVS → 尝试连接
  ├── 成功 → 绿灯 → 完成
  └── 失败 → 启动 AP + HTTP 配网服务器
              ├── 禁用 RTC 定时唤醒（配网期间不休眠）
              ├── 相册功能暂停（AP 模式检查）
              └── 手机连 PaperColor-XXXX → 打开 192.168.4.1
                    ↓
              用户填写配置，提交
                    ↓ handle_post_config
              1. 保存到 NVS slot 0
              2. 保存到 SD /sd/album/config.txt（始终写入 auth 字段）
              3. 返回 HTTP 200 "Saved successfully"
              4. 创建延迟连接任务
              5. Handler 返回
                    ↓
              延迟任务 (500ms)
              ├── wifi_prov_stop()     → 关闭 HTTP 服务器
              ├── wifi_mgr_connect_sta → 连接（自动切换 STA 模式）
              │     ├── LED 蓝呼吸 → 绿灯（成功）
              │     └── LED 蓝呼吸 → 红灯（失败）
              ├── 展示 5 秒
              ├── led_off()           → 关灯，相册恢复
              └── vTaskDelete(NULL)
```

### 关键设计决策

| 决策 | 原因 |
|------|------|
| 不开启 DNS 劫持 | 用户通过 AP IP 192.168.4.1 访问即可 |
| 延迟任务关闭 HTTP | 避免 `httpd_stop()` 在 handler 中死锁 |
| 不显式停止 AP | `sta_connect_slot()` 设 STA 模式时自动关闭 AP |
| 配网时禁用 RTC 唤醒 | 防止配网过程中被意外唤醒打断 |
| AP 模式下暂停相册 | 防止翻页、刷新、休眠干扰配网 |

## 存储模型

### NVS ("wifi" namespace)

每个 slot 存储字段：

| Key | 格式 | 示例 |
|-----|------|------|
| `ssid_%d` | string | `"MyWiFi"` |
| `pass_%d` | string | `"password"` |
| `auth_%d` | string | `"psk"` 或 `"enterprise"` |
| `identity_%d` | string | `"user@domain.com"` |
| `username_%d` | string | `"user@domain.com"` |

向后兼容：无 `auth` 字段的旧 slot 默认视为 `"psk"`。

### SD config.txt（/sd/album/config.txt）

```
ssid=MyEnterprise
auth=enterprise
identity=user@domain.com
username=user@domain.com
pass=eap_password
dns=114.114.114.114
updated=20260707
```

- `auth` 省略 → 默认 PSK（兼容旧文件）
- 保存时**始终写入** `auth` 字段，避免旧值残留
- `identity`/`username` 省略 → 用对方代替

### WiFi 配置保存函数

`wifi_save_config_to_sd()`（在 `wifi_provisioning.cpp` 中）：
- 从 NVS slot 0 读取配置
- 写入 SD /sd/album/config.txt
- 始终写入 `auth` 字段（PSK 或 Enterprise），覆盖旧值
- 由配网 handler 和相册 `save_wifi_to_sd()` 委托调用

### 常见失败原因

| 现象 | 根因 | 解决 |
|------|------|------|
| `reason=23` 极快断开（~300ms），TLS Alert | mbedTLS 不支持 TLS 1.0 | 切换到 internal TLS（Kconfig patch） |
| `reason=23` 中等速度断开（~1-5s） | EAP 凭证错误或 MSCHAPv2 认证被拒 | 核对 identity/username/password |
| 配网后立刻休眠 | AP 停止后连接开始前有空档 | `wifi_mgr_stop_ap()` 移到延迟任务中 |
| SD 读成 Enterprise（实际是 PSK） | 写 SD 时没覆盖旧 `auth=enterprise` | 始终写入 `auth` 字段 |
| HTTP handler 卡死 | `httpd_stop()` 在 handler 内调用 | 移至延迟任务 |
