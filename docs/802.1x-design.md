# 802.1x (PEAP-MSCHAPv2) 支持设计

## 概述

为 PaperColor 添加 WPA2-Enterprise 网络支持，使用 PEAP + MSCHAPv2 认证方式。
跳过服务器证书验证，不涉及 CA / 客户端证书，核心只需要 identity + username + password 三个新字段。

## 关键发现：TLS 1.0 兼容性

**某些企业 AP（如华为）的 PEAP 隧道只支持 TLS 1.0。**

```
Mac Client Hello:  Version TLS 1.2 (0x0303)
Server Hello:      Version TLS 1.0 (0x0301), cipher TLS_RSA_WITH_AES_128_CBC_SHA
Mac 接受降级 → 握手成功
```

ESP-IDF v5.5 使用 **mbedTLS 3.x**，该版本已**完全移除 TLS 1.0/1.1 代码**。
使用 mbedTLS 时，ESP32 会直接回复 `TLS Fatal Alert: Protocol Version`，不会尝试 Client Hello。

### 解决方案：wpa_supplicant 内部 TLS

ESP-IDF 的 wpa_supplicant 组件内置了独立的 TLS 实现（`tlsv1_client.c`），支持 TLS 1.0/1.1/1.2。
通过关闭 `ESP_WIFI_MBEDTLS_TLS_CLIENT` 选项启用：

```c
// tlsv1_common.h - 默认 TLS 版本
#define TLS_VERSION_1   0x0301  // TLSv1.0
#define TLS_VERSION_1_1 0x0302  // TLSv1.1
#define TLS_VERSION_1_2 0x0303  // TLSv1.2

// 未定义 CONFIG_TLSV12 时，默认使用 TLS_VERSION_1 (1.0)
#ifdef CONFIG_TLSV12
#define TLS_VERSION TLS_VERSION_1_2
#elif defined(CONFIG_TLSV11)
#define TLS_VERSION TLS_VERSION_1_1
#else
#define TLS_VERSION TLS_VERSION_1
#endif
```

ESP-IDF v5.5 的 Kconfig 不定义 `CONFIG_TLSV12` / `CONFIG_TLSV11`，因此内部 TLS 默认使用 TLS 1.0。

## 依赖

### 组件依赖（CMakeLists.txt）

```cmake
REQUIRES
    ...
    wpa_supplicant    # 提供 esp_eap_client.h
```

### 头文件

```c
#include <esp_eap_client.h>
```

### 关键 API

```c
// 设置身份信息（必须在 esp_wifi_sta_enterprise_enable() 之前调用）
esp_eap_client_set_identity((const uint8_t*)identity, len);
esp_eap_client_set_username((const uint8_t*)username, len);
esp_eap_client_set_password((const uint8_t*)password, len);

// 设置 EAP 方法
esp_eap_client_set_eap_methods(ESP_EAP_TYPE_PEAP);

// 禁用证书时间检查（RTC 可能未设置）
esp_eap_client_set_disable_time_check(true);

// 启用 Enterprise 模式
esp_wifi_sta_enterprise_enable();
```

### 跳过服务器证书验证

**不调用** `esp_eap_client_set_ca_cert()` 即可。（不需要传 NULL, 0）

## 构建配置：internal TLS

必须在 `sdkconfig` 中关闭 mbedTLS TLS client，强制 wpa_supplicant 使用内部 TLS：

```kconfig
# CONFIG_ESP_WIFI_MBEDTLS_TLS_CLIENT is not set
```

### 方法一：修改组件 Kconfig（推荐，避免被覆盖）

```bash
# 将 Kconfig 的 default y 改为 default n
sed -i 's/default y/default n/' \
  /opt/esp/idf/components/esp_wifi/Kconfig
```

每次重新创建容器或升级 IDF 后需重新执行。

### 方法二：编译后手动覆盖 sdkconfig

```bash
idf.py reconfigure
sed -i 's/CONFIG_ESP_WIFI_MBEDTLS_TLS_CLIENT=y/\
# CONFIG_ESP_WIFI_MBEDTLS_TLS_CLIENT is not set/' sdkconfig
cd build && ninja    # 跳过 cmake reconfigure
```

注意：再次运行 `idf.py build` 会因为 cmake reconfigure 覆盖掉这个改动。

## 连接流程

### Enterprise 连接步骤（精确顺序）

参考 ESP-IDF 官方例程 `wifi_enterprise_main.c`：

```c
// 1. 设置 WiFi 模式 + 配置
esp_wifi_set_mode(WIFI_MODE_STA);
esp_wifi_set_config(WIFI_IF_STA, &wc);  // wc.sta.ssid = "SSID"

// 2. 设置所有 EAP 参数
esp_eap_client_set_identity(...);
esp_eap_client_set_username(...);
esp_eap_client_set_password(...);
esp_eap_client_set_eap_methods(ESP_EAP_TYPE_PEAP);
esp_eap_client_set_disable_time_check(true);

// 3. 启用 Enterprise
esp_wifi_sta_enterprise_enable();

// 4. 启动 WiFi（触发 STA_START → event handler 调用 esp_wifi_connect()）
esp_wifi_start();
```

注意：
- EAP 参数**必须在** `esp_wifi_sta_enterprise_enable()` 之前设置
- `esp_wifi_sta_enterprise_enable()` **必须在** `esp_wifi_start()` 之前调用
- 不要设置 `wifi_config_t.sta.password`（EAP 客户端独立处理密码）
- 不要在 `wifi_config_t.sta.threshold.authmode` 设置 Enterprise 标志（wpa_supplicant 自动检测）

### 断开处理

```c
void wifi_mgr_disconnect_sta(void)
{
    esp_wifi_sta_enterprise_disable();  // 清除 EAP 状态
    esp_wifi_disconnect();
    esp_wifi_set_mode(WIFI_MODE_NULL);
}
```

## 存储模型

### NVS ("wifi" namespace)

每个 slot 增加字段：

| Key | 格式 | 示例 |
|-----|------|------|
| `auth_%d` | string | `"psk"` 或 `"enterprise"` |
| `identity_%d` | string | `"user@domain.com"` |
| `username_%d` | string | `"user@domain.com"` |
| `pass_%d` | string | 已有，存 EAP 密码 |

向后兼容：无 `auth` 字段的旧 slot 默认视为 `"psk"`。

### SD config.txt

```
ssid=MyEnterprise
auth=enterprise
identity=user@domain.com
username=user@domain.com
pass=eap_password
dns=114.114.114.114
updated=20260707
```

- `auth` 省略 → 默认 PSK（兼容旧文件）
- `identity` 省略 → 用 username 代替
- `username` 省略 → 用 identity 代替

### Provisioning 页面

配置 HTML 页面增加：

1. **认证类型** 下拉框：`PSK` / `Enterprise (PEAP)`
2. **EAP Identity** 输入框（Enterprise 时显示）
3. **Username** 输入框（Enterprise 时显示）

`/api/config` POST 解析新字段：

```json
{
  "ssid": "...",
  "auth": "enterprise",
  "identity": "user@domain.com",
  "username": "user@domain.com",
  "pass": "..."
}
```

兼容旧 POST（无 `auth` 字段 → `auth = "psk"`）。

### 常见失败原因

| 现象 | 根因 | 解决 |
|------|------|------|
| `reason=23` (IEEE8021X_FAILED) 极快断开 (~300ms) | mbedTLS 不支持 TLS 1.0，回复 Protocol Version Alert | 切换到 internal TLS |
| `reason=23` 中等速度断开 (~1-5s) | EAP 身份/密码错误，或 MSCHAPv2 认证被拒 | 核对凭证 |
| 连接卡在 `run` 状态 30s 后超时 | TLS 握手进行中但未完成（可能是证书验证） | 检查 `disable_time_check` 和 CA 证书设置 |
